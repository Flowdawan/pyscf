5.4. Density functional theory (DFT)¶

Modules: dft, pbc.dft

5.4.1. Overview¶

The KS-DFT methods in modules pyscf.dft and pyscf.pbc.dft are implemented as derived classes of the base SCF class pyscf.scf.hf.SCF. The major modifications of DFT with respect to HF include the re-implementation of the get_veff() and energy_elec() methods. In addition to get_jk(), get_veff() also calls numint.nr_rks() (for spin-restricted cases) or numint.nr_uks() (for spin-unrestricted cases) to compute the exchange-correlation (XC) energy and potential. The XC energy is added to the total electronic energy in energy_elec().

The key attributes of the KS classes include

xc

names of the XC functionals

nlc

names of the non-local correlation functionals

omega

\(\omega\) of the range-separated Coulomb operator \(e^{-\omega r_{12}^2} / r_{12}\)

grids

dft.gen_grid.Grids (or pbc.dft.gen_grid.Grids) objects which define the DFT grids

5.4.2. Numerical grids¶

The XC functionals are evaluated on numerical quadrature grids. These grids are defined in modules dft.gen_grid and pbc.dft.gen_grid for molecules and solids, respectively. Both uniform grids and Becke (atomic) grids are supported.

The base class for atomic grids is pyscf.dft.gen_grid.Grids, which include the following key attributes

level

grid level to control the number of radial and angular grids

atomic_radii

atomic radii, can be dft.radi.BRAGG_RADII (Bragg-Slater radii), dft.radi.COVALENT_RADII (covalent radii) or None (no atomic size adjustment in the atomic partition scheme of Becke)

radii_adjust

functions for atomic size adjustment, can be dft.radi.becke_atomic_radii_adjust(), dft.radi.treutler_atomic_radii_adjust() or None

radi_method

functions for radial grid schemes, can be dft.radi.treutler(), dft.radi.delley(), dft.radi.mura_knowles() or dft.radi.gauss_chebyshev()

becke_scheme

functions for atomic partition schemes, can be dft.gen_grid.original_becke() or dft.gen_grid.stratmann()

prune

functions for grid pruning, can be dft.gen_grid.nwchem_prune(), dft.gen_grid.sg1_prune(), dft.gen_grid.treutler_prune() or None (no pruning)

atom_grid

user defined number of radial and angular grids for specific atom types

coords

saved Cartesian coordinates of grid points

weights

saved weights of grid points

To generate the Becke grids, one can simply initialize the Grids object and then call the build() method:

g = Grids()
g.build()

Internally, build() calls the following functions:

  • pyscf.dft.gen_grid.gen_atomic_grids() – which generates the coordinates and weights of the grid points with respect to the atom center for each atom type

  • pyscf.dft.gen_grid.get_partition() – which generates the coordinates and weights of the molecular grid points

See examples/dft/11-grid_scheme.py for examples of specifying various grid schemes.

The PBC implementation of the Becke grids is defined in class pyscf.pbc.dft.gen_grids.BeckeGrids, which is derived from the base class pyscf.dft.gen_grid.Grids. The main modification of the PBC implementation is that the grid points contained in the reference unit cell (including those belonging to the periodic images) are all included. (see pyscf.pbc.dft.gen_grids.get_becke_grids())

The uniform grid is implemented in pyscf.pbc.dft.gen_grids.UniformGrids, whose build() method internally calls pyscf.pbc.gto.cell.get_uniform_grids().

5.4.3. Numerical integration¶

The actual methods to evaluate XC functionals and their related integrals are implemented in modules pyscf.dft.numint and pyscf.pbc.dft.numint. For example, the XC energy and potential matrix for a given density matrix are computed by nr_rks() (or nr_uks()), which internally calls

  • eval_ao() – to compute the atomic orbitals (AOs) and their derivatives on the grid

  • eval_rho() – to compute the electron density and density derivatives on the grid

  • eval_xc() – to compute the XC energy and potential through an interface to the external library Libxc (pyscf.dft.libxc.eval_xc()) or XCFun (pyscf.dft.xcfun.eval_xc()).

Other useful functions implemented in numint include

  • eval_mat() – evaluating the XC potential matrix with AO, electron density, and XC potential values on the grid as the input

  • nr_vxc() – evaluating the XC energy and potential matrix with the density matrix as the input

  • nr_sap_vxc() – evaluating the superposition of atomic potentials matrix, which is used as the initial guess for \(v_{\rm eff}\) when setting mf.init_guess to 'vsap'.

  • nr_rks_fxc(), nr_uks_fxc() – evaluating the XC kernel matrix
