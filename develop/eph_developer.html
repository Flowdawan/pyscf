5.15. Electron Phonon Coupling¶

Modules: eph, pbc.eph

5.15.1. Overview¶

All EPH classes in pyscf.eph are implemented as derived class of the base Hessian class, eg, pyscf.hessian.rhf.Hessian. These include

pyscf.eph.rhf.EPH

RHF

pyscf.eph.uhf.EPH

UHF

pyscf.eph.rks.EPH

RKS

pyscf.eph.uks.EPH

UKS

The key attributes of the EPH class include

cutoff_frequency

cutoff frequency in wavenumber unit

keep_imag_frequency

whether to store the imaginary modes

The main driver is the pyscf.eph.rhf.kernel() function which takes a EPH object and carries out the following steps:

  • building the nuclear hessian, which consists of

    • building the e-e part – hess_elec()

    • building the e-nuc part – hess_nuc()

  • computing the vibrational frequencies and polarization vectors – get_mode(), which calls solve_hmat() for diagonalization

  • building the electron phonon coupling matrix – get_eph(), which consists of

    • computing the e-nuc part – vnuc_generator()

    • computing the e-e part from MO derivative – rhf_deriv_generator()

    • computing the e-e part from orbital derivative – _get_jk() and additionally _get_vxc_deriv1() for dft

    • assembling the response matrix with eigenmodes/eigenvectors to the final matrix form

Internally, different methods reuse this kernel by overwriting the Hessian methods.

  • DFT SCF is implemented by specializing hess_elec() and hess_nuc() for the various KS objects (see e.g., pyscf.hessian.rks.hess_elec() and pyscf.hessian.rks.hess_nuc()).
